02 / Core workflow

Start with what matters. Lay the track as we go.

Business development is the likely first deep dive: use call notes, scope, and a representative slice of budget history to support a fully branded, human-reviewed HTML proposal and a more defensible early budget range, including a rough range that can be quoted live on a discovery call without boxing anyone in. Founder operating touchpoints and content scale are strong next candidates.

01 / Discover

Map the work

Create the priority list, walk the process with its owners, and inventory the tools, sources, decisions, and desired outcomes.

02 / Prioritize

Choose the target

Rank opportunities by value, repetition, feasibility, data readiness, effort, and risk.

03 / Design

Define the system

Inputs, controlled source access, expected outputs, Cardboard brand language, token cost, quality checks, ownership, and guardrails.

04 / Build

Implement the pilot

Create the selected workflow, test representative examples, and refine failure handling.

05 / Enable

Transfer ownership

Teach the team to run, inspect, approve, document, maintain, and improve the workflow.

Operating rule Discovery determines what gets built. No workflow is treated as committed until its human owner, source access, token and tool cost, handling of budget history or MNPI, and release gate are agreed.

08 / Working agreement

Clear boundaries keep the work moving.

Team size

Includes live enablement for up to six Cardboard Spaceship team members.

Client inputs

Cardboard Spaceship supplies representative call notes, proposals, the saved HTML proposal template and brand project file, and an agreed subset of historical line-item budgets (not the full archive), plus timely feedback.

Data handling

Client materials remain in client-approved tools. Budget history, public-company information, and MNPI are segmented or excluded unless an approved handling path is agreed in writing. Typical paths: code names for public-company work, and a data store Cardboard controls that the workflow queries for approved subsets instead of taking in the raw data.

Human approval

The workflow will not autonomously send pricing or make client commitments. A designated person reviews every release.

Implementation scope

After discovery, Design Innovations will identify what fits within the approved implementation hours and separate any future phase before additional work begins.

Retainer capacity

Retained work is tracked against the included monthly hours. Up to four unused implementation hours may roll forward one month; all other unused capacity resets.

Token + tool cost

Workflows are evaluated on recurring operating cost as well as output quality so a useful briefing or automation does not unexpectedly consume the team’s available capacity.

Not included

New software licenses, third-party usage fees, large-scale archive cleanup, custom enterprise integrations, production hosting, and work beyond the approved hours.

Offer validity

Pricing and reserved-capacity terms are valid through August 20, 2026.
